너는 시술 진단 웹앱을 만드는 시니어 프론트엔드 개발자야.
Next.js(App Router) + React + TypeScript로 “롯드 선정 진단기”를 만들어줘.
외부 DB 없이 로컬 상태로 동작하면 돼.

[기능]
- 사용자가 아래 3가지(필수) + 1가지(선택)를 선택한다.
  1) 뿌리 노출(덮임 정도): (덮인 곳 없음 / 조금 덮임·앞머리만 / 많이 덮임)
  2) 눈두덩 두께: (얇음 / 두꺼움)
  3) 뿌리 각도: (극하향 / 평균 / 극상향)
  4) (선택) “극하향모가 아닌 경우” 토글

- 선택값에 따라 추천 결과를 카드 형태로 출력한다:
  - 추천 컬: C컬 / 물방울 C컬 / L컬
  - 롯드 방향: 정방향 / 역방향(해당 시)
  - 사이즈: 다운 / 정 / 업
  - 시술 메모: 케이스별 안내 문구

[룰(우선순위)]
1) 뿌리 각도로 사이즈 결정:
   - 극하향: 사이즈 다운, 컬은 C컬 또는 L컬만 가능
   - 평균: 정사이즈
   - 극상향: 사이즈 업
2) 뿌리 덮임:
   - 덮임 없음: 컬 자유
   - 조금 덮임/앞머리만: 컬 자유 + 롯드 역방향 시술 옵션 표시
   - 많이 덮임: 물방울 C컬 꽉차게 또는 L컬을 추천 + 기본은 사이즈 업
     단, “극하향모가 아닌 경우” 토글이 켜져 있으면 사이즈는 1번(각도 룰)을 따른다.
3) 눈두덩 두꺼움이면: 결과 컬을 “바짝 C컬 또는 물방울 C컬” 쪽으로 보정(가능한 범위 내)

[요구]
- UI는 심플하고 모바일 친화적으로.
- 선택을 바꿀 때마다 결과가 즉시 갱신.
- 룰 엔진을 함수로 분리: getRecommendation(inputs) 형태.
- 전체 코드 파일 구조를 포함해 출력해줘.
